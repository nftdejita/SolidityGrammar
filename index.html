                                                    基本レイアウト

                                                    ​x
                                                    // SPDX-License-Identifier: MIT
                                                    pragma solidity ^0.8.17;
                                                    

                                                    contract HelloWorld {
                                                        string public greet = "Hello World!";
                                                    }

                                                     

                                                    • １行目のライセンス識別子は必須

                                                    • コメントは　//　もしくは　/* */ 　が利用可能

                                                    • ２行目の pragmaでコンパイル可能なバージョンを指定する（ただしエラーチェックのみ）

                                                    • contract コントラクト名　{ } が実際のコントラクトを表す（複数記述可能）

                                                    文字列は定義はできるが、文字列操作の関数がないためSmart Contractでは文字列を使わないようにする。

                                                    リテラル記法

                                                    • 普通の数字は10進法を意味する

                                                    • 1.6e18 = 1600000000000000000

                                                    • 文字列はダブルクォーテーションもしくはシングルクォーテーションで囲む

                                                    • リテラルでアンダースコアは無視される　　1_000_000_000

                                                    • 1 ether = 1e18 1 gwei = 1e9

                                                    • second, minutes, hours, days, weeks は単位として利用可能

                                                     

                                                    可視性 Visibility

                                                    　ステート変数には可視性と呼ばれる利用範囲を宣言する必要がある。

                                                    　なお、publicは引数の値をメモリに保存し、externalはメモリ保存しないため、externalの方がgasの消費が低く済む。

                                                     

                                                    変数の宣言方法

                                                    xxxxxxxxxx
                                                        uint public count = 1;
                                                        型　　可視性　変数名　初期値

                                                     

                                                    構造体

                                                    structキーワードを利用して構造体を設定できる

                                                    xxxxxxxxxx
                                                        // 構造体の定義
                                                        struct ToDoStruct {
                                                            string text；
                                                            bool finished;
                                                        }
                                                        
                                                        // 実体の宣言
                                                        ToDoStruct[] public todos;
                                                        
                                                        todos.push(ToDoStruct({test:'sample', finished: false}));
                                                        
                                                        if(todos[0].finished) {
                                                            //.....
                                                        }

                                                     

                                                    配列

                                                    xxxxxxxxxx
                                                        uint[] public arr1;
                                                        uint[] public arr2 = [1,2,3];
                                                        uint[3] public arr3 = [1,2,3];
                                                        string public str1 = 'abcde';

                                                    • 配列の操作は push pop deleteを利用する

                                                    • 配列には構造体も指定できる

                                                    • 文字列は配列として定義できるが操作はできない。どうしてもやりたければbytes配列に変換して行う。

                                                    Mapping

                                                    　キー値を指定して内容を取り出すことができる特殊な配列で、アドレス毎の取引残高などを格納する場合に良く利用される。

                                                    xxxxxxxxxx
                                                        // アドレスをキーとしてuint型のデータをbalanceという名前で取り出せる
                                                        mapping(address => uint) public balance;
                                                        
                                                        balance[msg.sender] = msg.value;

                                                    注) 関数のパラメーターや戻り値として受け渡しはできない

                                                    関数の引数

                                                    • 関数の引数にはマッピング変数は使えない

                                                    • 関数の引数はすべてローカル変数となる。通常はアンダースコアを付けて区別する

                                                    • 引数、戻り値ともにデータ場所の指定が必要

                                                    ViewとPure

                                                    　Smart Contractは、ワールドワイドコンピューティングを基本としているため、実行時のリソース利用については’資源の有効活用’の意識が重要となる。このため関数にはあらかじめ属性が付与され、機能制限を明示することで不要なリソースを割かないようにする（＝実行時のコストが安くなる）。

                                                     

                                                    コントラクトの基本動作

                                                    　Solidity はJavaに似たオブジェクト言語であるため、コントラクトはまず初めにコンストラクターが動作する。また、関数自身をオーバーライドできるため、ViewやPureなどと同様のオリジナルな規制をすることができる。

                                                    xxxxxxxxxx
                                                    contract Sample {
                                                    

                                                        address public owner;
                                                        
                                                        constructor() {
                                                            onwer = msg.sender;
                                                        }
                                                        
                                                        modifier onlyOwner() {
                                                            require(msg.sender == onwer, "Access denied");
                                                        }
                                                        
                                                        function changeOnwer(address _newOnwer) public onlyOnwer {
                                                            owner = _newOwner;
                                                        }
                                                    

                                                    }

                                                    • constructor()でコントラクト作成時に、初期パラメータとしてクリエイトユーザーのアカウントをオーナーに設定する

                                                    • modifier()でオーナー以外のアクセスを制限するように属性を作成する

                                                    • changeOnwer関数に対してmodifierを適用してオーナー以外のアクセスを規制する属性を継承させる

                                                     

                                                    チェック処理 Error

                                                    　入力値やパラメーターなどのチェック方法としてrequireとrevertがある。

                                                    xxxxxxxxxx
                                                        // Requireの例
                                                        function testRequire(uint _amount) public {
                                                            require( _amount >= 3, 'Sorry Minimum amount is 3');
                                                            //...
                                                        }
                                                      
                                                        // Revertを使って独自のエラーコードを返却する例
                                                        error MinimumAmount(uint indata);
                                                        
                                                        function testRevert(uint _amount) public pure {
                                                            if (_amount >= 3) {
                                                                revert MinimumAmount(indata: _amount);
                                                            }
                                                        }

                                                    　Requireは手軽だが実際にはガスコストがかかる。Revertの場合はエラーを個別に作成することで、データ転送量を４バイトに押さえられるため、ガスコストを圧縮できる。

                                                     

                                                    イベント処理 Event

                                                    　スマートコントラクトはブロックチェーン上の取引に過ぎないため、実際に実行されるタイミングはリアルタイムではない（時に数分待つことも）。

                                                    　スマートコントラクトを発行するクライアントアプリケーションでは、Event機能を利用することで、スマートコントラクトの実行完了タイミングなどを知ることができる。

                                                    xxxxxxxxxx
                                                        event Log(address indexed sender, string message);
                                                    

                                                        function test() public {
                                                            emit Log(msg.sender, "Hello World!");
                                                        }

                                                    　上記の例ではコントラクトを実行したアカウントと'Hello world'の文字列が発行される。受け取り側では(web3ライブラリを介して)イベント処理を記述できる。

                                                     

                                                    支払い可能宣言　Payable

                                                    　Etherを受け取れる関数とアカウントは必ずPayable修飾子をつけて受け取りを明示する必要がある。

                                                    xxxxxxxxxx
                                                        function deposit() payable public {
                                                            // ...
                                                            payable(msg.sender).transfer(msg.value);
                                                        }

                                                     

                                                    Etherの送金方法

                                                    　関数としては send transfer call　の３種類あるが、send、transferにはガスコストの上限があるため、Callによる記述が推奨となっている。

                                                    xxxxxxxxxx
                                                        // transfer:エラーの場合はそこで中断
                                                        toAddress.transfer(msg.value);
                                                    

                                                        // send:エラーの場合はfalseが返ってくるので後処理が必要
                                                        bool sent = toAddress.send(msg.value);
                                                    

                                                        // call
                                                        (bool sent, ) = toAddress.call{value: msg.value}("");

                                                     

                                                    Etherの受信方法

                                                    　なおコントラクトがEtherを受け取る方法としては、コントラクト内の関数を呼び出す際にEtherを合わせて送金するパターンと、コントラクトの関数を指定せずに直接Etherを送金するパターンがある。

                                                    　前者であれば呼び出された関数内で処理を行えばよい。

                                                    　後者の場合は関数が指定されていない(=msg.dataがない)のでrecive()関数が呼び出される。

                                                    xxxxxxxxxx
                                                        // Function to receive Ether. msg.data must be empty
                                                        receive() external payable {}
                                                    

                                                        // Fallback function is called when msg.data is not empty
                                                        fallback() external payable {}
                                                    

                                                        // 関数で受け取る場合
                                                        function depo() public {
                                                            require(msg.value>3e18,"Sorry, minimum amount is 3ether");
                                                        }

                                                    そもそもの問題は全ての取引トランザクションに送金データを乗せることができる点にある。

                                                     

                                                    乱数

                                                    　スマートコントラクトは複数のサーバーで同時に動き、その結果が全て同じになることが必要であるため、本来的な意味での乱数は扱えない。そのため一般的にはブロック番号をもとにした疑似乱数が用いられる。

                                                    xxxxxxxxxx
                                                     uint rand = uint(keccak256(abi.encodePacked(block.timestamp,msg.sender)));

                                                    　なお上記では256バイトの整数が返却される。ここから一桁の整数を得たい場合は剰余算(rand%10)を用いる。

                                                     

                                                    コントラクトのアカウント

                                                    　コントラクトもユーザーと同じくアカウントが付与される。なので作成者はコントラクトの生成に必要なコストを支払うからといって、コントラクトに対して何か操作できるわけではない。プログラム上に必要な権限を設定しそれを正しく設定する（コンストラクタで設定する or Setter関数を作るなど）が必要となる。

                                                    xxxxxxxxxx
                                                    // SPDX-License-Identifier: MIT
                                                    pragma solidity ^0.8.3;
                                                    

                                                    contract Charity {
                                                        address public owner;
                                                        
                                                        constructor() {
                                                            owner = payable(msg.sender);
                                                        }
                                                        
                                                        function donate() public payable {}
                                                    

                                                        function withdraw() public {
                                                            uint amount = address(this).balance;
                                                            owner.transfer(amount);
                                                        }
                                                    }

                                                    　このコントラクトは募金コントラクトであるが、寄付関数であるdonate()を呼んで寄付した金額は、Charityコントラクトのアドレスに紐づけられる。このため、寄付を受け取るにはownerとしてアカウントを登録する必要がある。
